"# DreamCatcher.AI Server\n+\n+The `server_ai` workspace powers the backend API, authentication, MongoDB models, and the OpenRouter/MCP integration that drives the AI dream interpretation layer.\n+\n+## Stack highlights\n+- **Runtime:** Node.js 20\n+- **Framework:** Express + TypeScript\n+- **Database:** MongoDB with Mongoose models (`dream`, `user`)+ Zod validation in `validation/`\n+- **AI middleware:** OpenRouter/MCP provider located under `llm/`, with helper hooks for language models\n+- **Infrastructure:** JWT auth, Nodemailer helpers, validation middleware, and error sanitization for production readiness\n+\n+## Quickstart\n+```bash\n+cd server_ai\n+npm install\n+npm run dev        # nodemon + tsc-watch\n+# For production build:\n+npm run build && npm start\n+```\n+Development runs on `http://localhost:3000` by default.\n+\n+## Environment (`server_ai/.env`)\n+```\n+PORT=3000\n+MONGO_URI=mongodb://localhost:27017/dreamcatcher\n+JWT_SECRET=replace-with-secret\n+APP_URL=http://localhost:5173\n+\n+# Optional LLM/OpenRouter keys\n+OPENROUTER_API_KEY=sk-...\n+OPENROUTER_MODEL=meta-llama/...\n+```\n+Add monitoring, analytics, or extra API credentials only when they are configured and stored securely.\n+\n+## Scripts\n+| Script | Description |\n+| --- | --- |\n+| `npm run dev` | Development server with `tsc-watch` + `nodemon`. |\n+| `npm run build` | Compiles TypeScript to `dist/`. |\n+| `npm start` | Runs the compiled server from `dist/`. |\n+| `npm test` | Jest unit tests (Mongo Memory Server). |\n+\n+## Directory overview\n+```\n+controllers/    # auth.controller.ts, dream.controller.ts, etc.\n+services/        # dream/service orchestrations, user flows\n+models/          # Mongoose schemas for users, dreams, and shared content\n+validation/      # Zod schemas used by request middleware\n+routes/          # Express route definitions wired to Layout\n+llm/             # OpenRouter provider, MCP client, LLM types\n+mcp/             # MCP client + helpers\n+helpers/         # mailer, error helpers, bcrypt utils\n+types/           # shared TS definitions (DreamCategory, responses, etc.)\n+```\n+\n+## Production guidance\n+1. Supply a secure `JWT_SECRET` and point `MONGO_URI` at your production database.\n+2. Enable TLS/HTTPS via reverse proxy (e.g., Nginx or cloud load balancer) before exposing the API.\n+3. Run `npm run lint`, `npm test`, and `npm run build` before rolling out a release.\n+4. Configure rate limiting, Redis session caching, or Sentry tracing as needed; the scaffolding supports middleware extensions.\n+5. Use the `APP_URL` variable so the backend can craft full URLs (emails, redirects, google callbacks).\n+\n+This backend is shaped for production: typed contracts, central validation, AI orchestration, and clean deployment scripts."
